# 志愿活动管理与积分激励平台

本项目是“高级软件工程”课程期末作业，主题为志愿活动管理与积分激励平台。系统围绕志愿活动发布、报名审核、签到签退、积分发放、积分兑换和后台数据管理等场景展开，面向管理员和志愿者两类用户提供完整的前后端功能。

## 项目概述

平台通过积分激励机制鼓励用户参与志愿活动。管理员可以维护活动、用户、报名、签到、积分和兑换记录；志愿者可以浏览活动、报名参与、完成签到签退、查看个人积分并兑换奖励商品。

## 技术栈

- 前端：Vue 3、Vue Router、Pinia、Element Plus、Axios、Vite
- 后端：Spring Boot 3.2、MyBatis-Plus、MySQL、JWT
- 数据库：MySQL，默认数据库名为 `volunteer_db`
- 构建工具：npm、Maven


## 快速启动

项目根目录下提供了一键脚本。

启动：

```bash
./start.sh
```

停止：

```bash
./stop.sh
```

默认访问地址：

- 前端页面：`http://127.0.0.1:3007/`
- 后端接口：`http://127.0.0.1:8087/`

`stop.sh` 默认只停止前端和后端服务，不停止 MySQL。如果需要同时停止 MySQL：

```bash
STOP_MYSQL=1 ./stop.sh
```

## 功能模块说明

### 1. 认证模块

认证模块负责系统登录、注册、登出和当前用户信息获取。用户登录成功后，后端签发 JWT Token，前端将 Token 存储在本地，并在后续请求中通过 `Authorization` 请求头携带，用于访问受保护接口。

主要功能：

- 志愿者注册
- 用户登录
- 用户登出
- 当前登录用户信息获取
- 登录状态校验和路由权限控制

### 2. 用户管理模块

用户管理模块用于维护平台用户信息。普通志愿者可以查看和修改自己的基础资料；管理员可以查看用户列表、按条件筛选用户、编辑用户信息、启用或禁用用户账号，并删除不再需要的用户账号。

主要功能：

- 个人信息查看与修改
- 修改密码
- 用户列表查询
- 用户角色区分
- 用户启用和禁用
- 管理员删除用户
- 删除用户时禁止删除当前登录的管理员账号
- 用户积分、可用积分、志愿时长展示

### 3. 活动管理模块

活动模块是系统的核心模块，用于管理志愿活动从发布到结束的完整生命周期。管理员可以创建、编辑、删除和调整活动状态；志愿者可以浏览活动大厅、查看活动详情并选择报名。

主要功能：

- 活动创建
- 活动列表查询
- 活动状态筛选
- 活动详情展示
- 活动编辑和删除
- 活动状态管理
- 活动人数、积分、时长和地点展示

活动状态包括：

- 草稿
- 报名中
- 进行中
- 已结束
- 已取消

### 4. 报名审核模块

报名模块连接志愿者和活动。志愿者可以对报名中的活动提交报名申请，也可以取消已报名活动；管理员负责审核报名记录，决定报名是否通过。

主要功能：

- 志愿者报名活动
- 志愿者取消报名
- 我的报名记录查询
- 管理员查看报名列表
- 管理员通过或拒绝报名
- 活动报名人数同步维护

### 5. 签到签退模块

签到模块用于记录志愿者实际参与活动的情况。志愿者在活动进行中可以签到，活动结束或服务完成后进行签退；系统根据签到签退记录计算实际志愿时长和可获得积分。

主要功能：

- 活动签到
- 活动签退
- 实际服务时长记录
- 完成后自动计算积分
- 我的签到记录查询
- 管理员查看签到记录
- 管理员补签

### 6. 积分管理模块

积分模块用于记录志愿者积分变化。积分来源主要包括活动完成奖励、管理员调整；积分消耗主要来自积分商城兑换。系统保存每一笔积分变动，方便追踪和展示。

主要功能：

- 我的积分统计
- 积分明细查询
- 积分排行榜
- 活动完成后积分增加
- 商品兑换后积分扣减
- 管理员手动调整积分
- 积分余额记录

### 7. 积分商城模块

积分商城模块用于展示可兑换奖励商品。志愿者可以用可用积分兑换商品；管理员可以维护商品信息，包括商品名称、描述、库存、所需积分和上下架状态。

主要功能：

- 上架商品展示
- 商品库存展示
- 所需积分展示
- 志愿者提交兑换申请
- 管理员新增商品
- 管理员编辑商品
- 管理员删除商品
- 商品上架和下架

### 8. 兑换管理模块

兑换模块记录志愿者的积分兑换行为。志愿者可以查看自己的兑换记录；管理员可以查看全部兑换申请，并更新兑换处理状态。

主要功能：

- 志愿者兑换商品
- 我的兑换记录查询
- 管理员查看兑换列表
- 兑换状态管理
- 兑换后扣减积分
- 商品库存和兑换次数维护

兑换状态包括：

- 待发放
- 已发放
- 已取消

### 9. 数据统计模块

统计模块用于展示平台运行情况和用户参与情况。管理员可以查看平台概览数据和活动参与排行；志愿者可以查看自己的积分、时长和参与表现。

主要功能：

- 平台总用户数统计
- 平台总活动数统计
- 总志愿时长统计
- 活动参与排行
- 志愿者个人积分统计
- 志愿者个人时长统计
- 积分排行榜

### 10. 前端页面模块

前端按角色划分为管理员端和志愿者端。登录后系统根据用户角色自动跳转到对应页面，并通过路由守卫控制访问权限。

管理员端页面：

- 数据概览
- 活动管理
- 报名管理
- 签到管理
- 用户管理：支持查询、编辑、启用、禁用和删除用户
- 商品管理
- 兑换管理
- 积分管理
- 个人中心

志愿者端页面：

- 活动大厅
- 活动详情
- 我的活动
- 我的积分
- 积分商城
- 我的兑换
- 排行榜
- 个人中心

## 数据库说明

系统使用 MySQL 数据库，初始化脚本位于：

```text
backend/src/main/resources/sql/init.sql
```

主要数据表：

- `user`：用户表
- `activity`：志愿活动表
- `enrollment`：活动报名表
- `attendance`：签到记录表
- `points_record`：积分记录表
- `reward`：奖励商品表
- `exchange_record`：兑换记录表

默认数据库连接配置位于：

```text
backend/src/main/resources/application.yml
```

默认配置：

- 数据库：`volunteer_db`
- 地址：`localhost:3306`
- 用户名：`root`
- 密码：`1234`

## 常用命令

前端单独启动：

```bash
cd frontend
npm install
npm run dev
```

后端单独启动：

```bash
cd backend
JAVA_HOME=/opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk/Contents/Home \
PATH=/opt/homebrew/opt/openjdk@17/bin:/opt/homebrew/bin:$PATH \
mvn spring-boot:run
```

后端打包：

```bash
cd backend
JAVA_HOME=/opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk/Contents/Home \
PATH=/opt/homebrew/opt/openjdk@17/bin:/opt/homebrew/bin:$PATH \
```

```bash
python3 clean_macos_junk.py . --delete
```

## 说明

如果访问 `http://127.0.0.1:8087/` 出现 Spring Boot Whitelabel 404 页面，这是正常现象。后端端口只提供接口服务，网页应访问前端地址 `http://127.0.0.1:3007/`。
